    breakpoints

    $breakpoints: ();

    Description

    Map of available breakpoints.

    Type

    Map

    Example

    $breakpoints: (
    small: "(max-width: 500px)",
    big:   "(min-width: 2000px)"
);

    Used by

    Author

    mixins

    respond-to

    @mixin respond-to($min: 0, $max: 0) { ... }

    Description

    Breakpoint mixin to handle responsive views.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $min

    Units are required for numerical parameters.

    		String or Number0
    $max

    Units are required for numerical parameters.

    		Number0

    Content

    This mixin allows extra content to be passed (through the @content directive). Role: Styles that should be modified in responsive view

    Example

    @include response-to('small') {
    // your responsive styles ...
}

    Throws

    • No value found for #{$breakpoint}.

    Requires

    • [variable] breakpoints
    • [function] get-breakpoint

    Used by

    • [mixin] response-to

    Author

    • Martin Rocek

    mixins

    font-size

    @mixin font-size($fs-map, $fs-breakpoints: $breakpoints) { ... }

    Description

    Font-size mixin that calculates accurate font-size and line-height to available breakpoints.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $fs-map—noneMap—none
    $fs-breakpoints—noneMap$breakpoints

    Example

    $fs-map variable example

    $p-sizes: (
    null: (1em, 1.3), // null field should be always present
    small: 1.1em,
    medium: (1.2em, 1.4),
    900px: 1.5em, // it's possible to use min viewport width as key too
)

    usage example

    p {
    @include font-size($p-sizes);
}

    Requires

    • [variable] breakpoints
    • [mixin] make-font-size

    Author

    • Martin Rocek
